Executive Summary
Hammer Voice Explorer (HVE) is an automated IVR discovery and documentation tool designed to solve the "black box" problem of legacy contact centers. It utilizes machine learning to navigate IVR menus automatically, creating accurate maps of the customer journey and generating test scripts for regression libraries. It is the primary tool for "auditing what exists" before a migration and validating design adherence after deployment.
________________________________________
1. Business Cases (Strategic Drivers)
•	Accelerating Cloud Migrations (Migration Readiness):
o	Driver: Enterprises often delay migrations because they lack documentation for their legacy platforms ("I have an IVR but don't know what applications are on it").
o	Solution: HVE provides a "snapshot of what exists in the legacy environment" , enabling "more efficient cloud IVR design and buildout". This prevents the migration team from rebuilding "in the dark."
•	Operational Efficiency (QA Productivity):
o	Driver: Creating test scripts manually is slow, leading to "productivity loss" and bottlenecks in User Acceptance Testing (UAT).
o	Solution: By automatically generating scripts from the discovery process, HVE helps "reduce 80% [of] Test Automation Development" , significantly shortening testing cycles.
•	Risk Mitigation & Audit (Shadow IT):
o	Driver: Over time, IVR menus change without updates to documentation ("developers fail to provide update on changes").
o	Solution: HVE allows IT leaders to "audit what exists against all these toll-free numbers", ensuring that the documented design matches the actual production reality.
•	Customer Experience Optimization:
o	Driver: Broken links or dead ends in an IVR frustrate customers.
o	Solution: The "negative test discovery" feature identifies navigation errors and potential dead ends, allowing teams to optimize the menu flow before customers encounter issues.

2. Use Cases (Operational Scenarios)
A. Automated Discovery & Mapping
•	The "Digital Twin" Blueprint:
o	Scenario: An enterprise has a legacy Avaya IVR with 10 years of undocumented changes.
o	Execution: HVE automatically navigates the system to "discover and document" the current state , producing a visual map or diagram that business teams can use to collaborate.
•	Migration Design Verification:
o	Scenario: After building the new IVR in the cloud (e.g., Genesys/Amazon Connect), IT needs to prove it matches the requirements.
o	Execution: "Discover and document your new cloud IVR to verify its design adherence" against the original legacy maps.
B. Test Script Generation
•	Auto-Populating Regression Libraries:
o	Scenario: QA teams need to build a regression suite for the new release but lack time.
o	Execution: HVE converts the discovered paths into executable test scripts. These "automatically generated scripts can then be leveraged... for ongoing regression testing", replacing manual script writing.
•	Persona-Driven Testing:
o	Scenario: Verifying that a "Platinum" customer gets different menu options than a "Delinquent" customer.
o	Execution: Configure HVE with specific inputs (account number, PIN) to "identify a persona-driven testing approach", verifying that dynamic routing logic works correctly.
C. Maintenance & Optimization
•	Negative Testing (Error Handling):
o	Scenario: identifying "ghost" menu options that lead nowhere.
o	Execution: Use the "negative test discovery" capability to specifically hunt for "navigation errors" or unhandled inputs.

o	Metric: Reduction in hours spent writing test code.

•	Documentation Effort Reduction:
o	Metric: Hours saved creating Visio flows/documentation.

•	UAT Cycle Time:
o	Metric: Reduction in weeks/days required for UAT.
o	ROI Logic: By auto-generating the scripts and verifying the design upfront, enterprises "shorten user acceptance testing (UAT) timelines".
•	Migration Delay Avoidance:
o	Metric: Reduction in project delays caused by "discovery" phases.
o	ROI Logic: Eliminates the manual "human error and subjective interpretation" of legacy systems, preventing costly scope creep during the build phase.
